# MQAR 实验计划

## 1. 目标与指标
**目标**：使用多查询关联回忆（MQAR）任务评估线性注意力模型（RelaxedKaczmarzQNorm, RelaxedKaczmarz, Longhorn, Kaczmarz, GDN）的关联回忆能力。

**核心指标**：
- **准确率（Accuracy）**：查询值的正确检索百分比（掩码标记预测）。
- **收敛速度（Convergence Speed）**：达到 99% 准确率（或平台期）所需的步数。
- **记忆容量（Memory Capacity）**：随序列长度变化的性能表现。

**基线**：
- **随机猜测（Random）**：1 / Vocab_Size（约 0.01%）。
- **理论最优（Optimal）**：100% 准确率（合成任务是确定性的）。

## 2. 数据集构建协议
严格遵循 arXiv 2312.04927。
- **生成器**：`mqar_data.py`
- **词表大小**：8192 token。
- **划分**：8:1:1（训练集/验证集/测试集）。
- **关联模式**：
  - 键值对（2-gram）：`k -> v`
  - 3-gram：`k1 k2 -> v`
  - 4-gram：`k1 k2 k3 -> v`
- **分布**：
  - 键/值均匀采样。
  - 查询位置使用幂律分布（alpha=0.1）放置，以测试跨距离的回忆能力。
- **噪声**：从噪声词表中采样的填充 token。

## 3. 模型适配方案
模型适配于小规模验证设置：
- **架构**：2 层，256 嵌入维度，4 个头。
- **输入**：标准嵌入层。
- **输出**：语言建模头（预测下一个 token）。
- **配置**：在 `lit_gpt/config.py` 中添加了 `*_MQAR` 配置。

## 4. 实验流程
- **超参数**：
  - 学习率：1e-3 (AdamW)
  - 批量大小：32
  - 最大步数：20k（足以在小任务上收敛）
- **早停机制**：验证集准确率连续 10 次评估无提升。
- **检查点保存**：每 1000 步保存一次，基于验证集准确率保留前 5 个。

## 5. 评估框架
- **指标**：仅针对检索目标的 Token 级准确率（忽略填充物/键）。
- **分析**：
  - `mqar_analysis.py`：从 JSON 日志中汇总结果。
  - 图表：准确率 vs 序列长度；准确率 vs 关联阶数。
  - 统计检验：分析中包含 T 检验比较。

## 6. 复现性
- **种子**：所有随机数生成器（numpy, torch）固定为 42。
- **环境**：Linux, PyTorch 2.x, Lightning Fabric。
- **脚本**：`run_mqar_experiment.sh` 执行端到端复现（数据生成 -> 训练 -> 评估）。

## 7. 交付物
- `mqar_data.py`：数据生成器。
- `train_mqar.py`：训练脚本。
- `run_mqar_experiment.sh`：复现脚本。
- `mqar_analysis.py` / `.ipynb`：分析工具。
- `lit_gpt/config.py`：模型配置。
